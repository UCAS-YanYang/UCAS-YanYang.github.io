Notes on Hexo

Posted on 2023-06-20 In Hexo
In this article, record some of the pitfalls encountered when using hexo

本文主要记录一些在使用Hexo部署博客时遇到的坑。

Hexo项目简介

  1. 安装 Hexo： Hexo 是一个基于 Node.js 的静态网站生成器，它可以将你编写的 Markdown 文件转换为静态网页。首先，你需要安装 Node.js，并使用 npm（Node.js 包管理器）来安装 Hexo。
  2. 创建 Hexo 项目： 在命令行终端中，通过运行 hexo init <folder> 创建 Hexo 项目，其中 <folder> 是你要创建的项目文件夹的名称。Hexo 将为你创建一个基本的项目结构，包括配置文件、主题和内容文件夹。
  3. 配置 Hexo： 进入 Hexo 项目文件夹，并编辑 _config.yml 文件来配置你的个人主页的设置，比如网站标题、描述、主题等。根据需要进行相应的定制化设置。
  4. 编写内容： 在 Hexo 项目的 source/_posts 目录中，你可以创建 Markdown 文件来编写博客文章或页面内容。使用 Markdown 格式编写的文件可以包含文本、图片、链接等。每当你添加、修改或删除文章时，Hexo 将重新生成静态网页。
  5. 生成静态网页： 在命令行终端中，运行 hexo generate 命令来生成静态网页。Hexo 将会解析你的 Markdown 文件，并根据配置生成相应的 HTML、CSS 和 JavaScript 文件，并将它们存储在 public 文件夹中。
  6. 部署到 GitHub： 将生成的静态网页部署到 GitHub Pages 上，使你的个人主页可以在线访问。在 GitHub 上创建一个新的仓库，并将其命名为 <你的用户名>.github.io。然后，将 Hexo 生成的 public 文件夹中的内容复制到该仓库的主分支上。
  7. 配置 GitHub Pages： 在 GitHub 仓库的设置中，找到 “GitHub Pages” 选项，并选择将 “Source” 设置为 “master branch”。保存设置后，GitHub Pages 将开始自动构建并托管你的个人主页。
  8. 更新和发布： 当你编写新的博客文章或修改现有内容时，重复步骤 4 到步骤 7，即编写内容、生成静态网页并部署到 GitHub Pages。

总结起来，Hexo 通过将 Markdown 文件转换为静态网页，以便在 GitHub 上托管和展示你的个人主页。你可以使用 Hexo 提供的命令进行生成和部署操作，而 GitHub Pages 则负责将生成的静态网页发布到互联网上，使其他人能够访问和浏览你的个人主页。

配置、使用说明（坑点）

  • 卸载npm装过的包时，需要use the syntax npm uninstall package-name in the directory where the package is located.

写作

  • Hexo 配置文件（_config.yml）中的一些配置项：

    • timezone: Asia/Shanghai，没有Beijing这个选项，写Beijing在部署的时候会报错

    • url: https://ucas-yanyang.github.io/，这个链接到你的主页网址

    • permalink: :title/，这里的:title为source/_post下的相对路径

    • post_asset_folder: true，这个在后面插图片的时候有用

    • deploy:
  type: git
  repository: git@github.com:UCAS-YanYang/UCAS-YanYang.github.io.git  #你的仓库地址
  branch: master

 hexo new [layout] "文章标题"
 或者
 hexo new [layout] --path Hexo_Notes/hexo_notes "文章标题"
 
 hexo generate
 
 hexo server
 
 hexo deploy
